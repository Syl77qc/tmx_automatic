Architecture du système TMX v2
Mean reversion sur 12 FNBs canadiens (TSX) · Paper trading · Phase 1 en cours
Flux de décision
yfinance / Questrade → Scanner z-scores → Agent Groq → Verdict → Gmail + SMS → Décision humaine → Disnat (manuel)
Principe fondamental : Le système détecte, analyse et notifie — la décision finale et l'exécution restent humaines. Le scanner tourne automatiquement toutes les 5 minutes pendant les heures de marché (9h25–17h05 HE, lundi–vendredi) via GitHub Actions.
Les 4 maillons du signal
MAILLON 1
Régime VIX
VIX < 16 : Risk-on — taille normale
VIX 16–25 : Neutre — taille ÷2
VIX > 25 : Risk-off — pause
MAILLON 2
Signal z-score
Z20 ≤ −2.0 é.-t. pour déclencher
Z60 confirme ou ajuste la taille
Calculé sur prix de clôture quotidiens
MAILLON 3
Filtres A–H
7 filtres de sécurité + Filtre H (XIT)
Cluster, régime, profil, SMA50,
volume, BdC, corrélation, publications
MAILLON 4
Exécution & sortie
Sortie partielle à 50% du rebond
Trailing stop 1.5%
Horizon max selon profil FNB
Filtres de sécurité (Maillon 3)
Filtre A — Cluster
4–6 signaux simultanés → taille ÷2. 7+ signaux → bloquer tout.
Filtre B — Régime
Risk-off (VIX > 25) → bloquer sauf signaux extrêmes sur défensifs.
Filtre C — Profil
Z-score doit dépasser le seuil effectif du profil FNB (2.0 à 3.0).
Filtre D — XIU
XIU stable = correction sectorielle isolée → seuil +0.5, taille ÷1.5.
Filtre E — Volume
Volume élevé lors de la baisse = capitulation = signal de meilleure qualité.
Filtre F — SMA50
Sous SMA50 → taille ÷2. Protège contre les tendances baissières structurelles.
Filtre G — Blocs
Max 1 position par bloc corrélé (marché large, métaux, taux).
Filtre H — Publications
XIT uniquement : seuil 3.0 ±2j Shopify, bloquer le jour J. (PRD v2.2)

Infrastructure technique
Données de marché : yfinance (source principale) avec Questrade API comme source temps réel quand disponible. VIX via yfinance (^VIX).
Agent contextuel : Groq llama-3.3-70b analyse les actualités RSS (Yahoo Finance + Google News) pour chaque signal et classifie la baisse en SECTORIEL / FONDAMENTAL / SYSTÉMIQUE.
Automatisation : GitHub Actions (cron toutes les 5 min, 9h25–17h05 HE). Notifications Gmail HTML + SMS via adresse email opérateur.
Paper trading : Simulateur JSON interne. Capital virtuel 100 000 $ CAD. Modèle de coûts réaliste : spread 0.05%/côté + commission 4.95$ + slippage 0.05%.
Avis important — Ce site ne constitue pas un conseil financier. FNBLab est un projet de recherche personnel. Toutes les positions sont du paper trading (simulation). Les performances passées ne garantissent pas les performances futures.
